## 1. 项目概述

**herness** 是一个**个人自用**的加密货币量化交易系统：从行情数据采集、策略开发、回测验证，
到风控保护下的实盘交易，配合前端可视化看板进行投研与监控。

- **定位**：自用量化研究 + 交易执行工具（非对外产品，单用户）
- **核心理念**：策略可复现、回测可信赖、实盘可控制（风控优先于收益）

## 2. 目标用户与场景

| 维度 | 说明 |
|------|------|
| 用户 | 单人（项目拥有者），既是开发者也是使用者 |
| 部署 | 本地/单机为主，后端 FastAPI + 前端 React 本地运行 |
| 使用流程 | 研究行情 → 写策略 → 回测 → 风控配置 → 测试网验证 → 主网小仓位实盘 → 看板监控 |

## 3. 交易所与品种

| 维度 | 选型 |
|------|------|
| 交易所 | **OKX**（V5 API） |
| 品种 | **现货（SPOT）** + **U本位永续合约（SWAP）** |
| 对接方式 | **直接调 OKX 官方 API/SDK**（不用 CCXT，减少抽象层与依赖风险） |
| 鉴权 | API Key + Secret + Passphrase，HMAC SHA256 签名；私钥**绝不入库**，走 `backend/app/core/config.py` + `.env` |

> OKX 术语：永续合约在 OKX 叫 **SWAP**（如 `BTC-USDT-SWAP`），现货如 `BTC-USDT`。

## 4. 数据存储

| 数据 | 选型 |
|------|------|
| 行情（K线/Ticker）、回测结果、策略参数、交易记录 | **SQLite**（单文件，轻量，自用足够） |
| 时序扩展 | 未来数据量大时可迁移 DuckDB/TimescaleDB，当前不提前优化 |

## 5. MVP 功能范围

第一版包含以下 7 个模块（按里程碑顺序推进，见 §8）：

| 模块 | 说明 |
|------|------|
| **行情数据** | OKX 公开行情：REST 拉历史 K 线 + WS 实时订阅；落 SQLite；支持多周期 |
| **策略开发** | 纯函数式策略接口：输入行情+参数，输出信号；参数化管理 |
| **回测引擎** | 历史 K 线驱动策略，产出绩效指标（收益/回撤/夏普/胜率）；回测即测试 |
| **风控** | 仓位/杠杆/回撤/亏损上限 + 全局 kill switch（见 §7） |
| **实盘交易** | OKX 交易 API：下单/撤单/查仓位/查账户；**测试网先行** |
| **因子研究** | 因子计算、因子收益/IC 分析 |
| **前端看板** | K 线图、回测结果、资金曲线、持仓与订单监控 |

**显式排除（v1 不做）**：多租户/账号体系、付费/计费、跨交易所套tick、高频做市（低延迟）。

## 6. 关键非功能需求

- **可复现性**：策略层零 IO（[backend/AGENTS.md](../backend/AGENTS.md) 约束），回测确定性，随机数用固定种子。
- **安全性**：API Key 仅存 `.env`，`.gitignore` 已排除；代码中禁止硬编码；提交前 `entropy-gardener.sh` 检查 `.env` 未入库。
- **可观测性**：每次下单/风控触发记日志；前端看板可看实时状态。
- **幂等与限流**：OKX 有请求频率限制，交易层需做幂等（client订单号）与限流。

## 7. 风控设计（实盘前的硬性门禁）

加密永续合约带杠杆，风控**必须在任何实盘下单之前就位并强制启用**：

| 风控项 | 说明 |
|--------|------|
| 单标的仓位上限 | 单币种最大持仓占比/张数 |
| 总仓位上限 | 全账户最大敞口 |
| 最大杠杆上限 | 强制低于 OKX 允许的杠杆上限 |
| 最大回撤止损 | 触及回撤阈值 → 停止开新仓 |
| 每日亏损上限 | 当日亏损达限 → 暂停交易 + 撤挂单 |
| 价格保护带 | 下单价格偏离标记价过远 → 拒单 |
| 下单频率限制 | 防误循环刷单 |
| **全局 kill switch** | 一键：停止所有策略、撤所有挂单、平仓（可选） |
| API Key 权限最小化 | 仅开"交易"权限，**绝不给"提现"权限** |

> 风控不是可选项：实盘模式下，任何下单都必须先过风控闸；风控禁用则禁止下单。

## 8. 里程碑路线图（安全序列）

> 实盘真金白银是高风险点，按"测试网先行 → 风控门禁 → 主网灰度"推进。

| 里程碑 | 内容 | 状态 |
|--------|------|------|
| **M0** | 驾驭层骨架 | ✅ 已完成 |
| **M1** | OKX 行情数据层：REST 历史 K 线 + WS 实时订阅 → SQLite | 待开始 |
| **M2** | 策略框架 + 回测引擎 + 回测即测试 | 待开始 |
| **M3** | 前端看板：K 线 / 回测结果 / 资金曲线 | 待开始 |
| **M4** | 风控引擎（§7 全部规则） | 待开始 |
| **M5** | OKX 交易对接：**demo 测试网**鉴权/下单/撤单/查仓位 | 待开始 |
| **M6** | 因子研究模块 | 待开始 |
| **M7** | 实盘主网：风控门禁通过后，最小仓位灰度切换 | 待开始 |

**顺序约束**：M4（风控）必须在 M5（交易下单）之前完成；M7（主网）必须 M5（测试网）验证通过且风控全绿才可开启。

> 注：M5 的 demo 测试网本质就是你跳过的"模拟盘"——我把它作为实盘的**强制前置门**折进路线图，因为对真钱实盘而言这一步不可省。OKX demo 通过请求头 `x-simulated-trading: 1` 切换，无需另起环境。
